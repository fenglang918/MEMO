# 读博 vs 直接工作（Plan）

> 说明：脱敏示例项目。公开模板版只保留框架与可复用的对比维度。

## 总体策略

- 先把“本质区别”说清楚（评价函数/产出形式/反馈周期/风险结构），避免用噪声维度（头衔、想象、短期情绪）做决定。
- 用“现实检验”替代纯脑内推演：访谈 + 小型试做（研究/工程）+ 约束对齐（财务/地域/健康/签证等）。
- 把决策写成可复盘的 ADR：包含假设、证据来源、权衡与反事实检查。

## 里程碑与阶段

### 阶段 1：框架澄清（本周）

- [ ] 完成“本质区别”对比稿：`cognition/001-phd-vs-job-essence.md`
- [ ] 列出需要验证的关键假设（例如：我对研究/工业的日常想象是否准确）

### 阶段 2：信息收集（接下来 2–4 周）

- [ ] 访谈：至少 3 位在读 PhD / 3 位工业从业者（同领域优先）
- [ ] 收集典型日程、评价机制、压力来源、导师/团队文化差异样本
- [ ] 梳理“最坏情况/退出成本/可逆性”并量化机会成本

### 阶段 3：现实试做（接下来 4–8 周）

- [ ] 研究试做：选一个可在 2–4 周内收敛的 research mini-project（写作/复现/小实验）
- [ ] 工程试做：选一个可在 2–4 周内交付的工程 mini-project（产品化/开源贡献/端到端交付）
- [ ] 记录体验差异：动机、心流、压力、反馈质量、可持续性

### 阶段 4：做出选择（截止日期：____）

- [ ] 写最终决策 ADR：`decisions/001-choose-path.md`（待建）
- [ ] 将决策落到下一步行动清单（申请/找工作/作品集/推荐信/导师沟通等）

## 风险与应对

- 风险：信息来源偏差（只听到“幸存者叙事”）；应对：覆盖不同阶段与不同结果的人，记录样本偏差。
- 风险：把“身份/面子”误当成目标；应对：先写清楚目标函数（我到底要什么），再看路径匹配度。
- 风险：拖延导致错过窗口；应对：设置明确截止日期与最小可行动方案。
