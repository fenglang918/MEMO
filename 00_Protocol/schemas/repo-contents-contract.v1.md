# Repo Contents Contract v1

## 1. 目标

定义这个模板仓库里“什么内容放在哪里、为什么这样放”，作为：

- 人写文档时的结构约束
- Agent 检索 / 引用 / 回链时的语义依据
- 可选检索工具的上游契约

联邦导航入口：

- [`protocol-federation-map.v1.md`](protocol-federation-map.v1.md)

## 2. 顶层内容域语义

- `00_Protocol/`：contents 协议与使用约定
- `01_Inbox/`：待处理输入与临时收集
- `02_Desires/`：上游驱动（价值观 / 偏好 / 欲望）
- `03_Goal-Projects/`：目标与项目主域
- `04_Assets/`：稳定资产与个人资料
- `05_Resources/`：外部可调用资源
- `06_Infra/`：脚本、插件、i18n、工具与实验层
- `07_Principles/`：跨项目原则与裁决约束
- `08_Operations/`：运营与日常运作记录
- `develop/`：模板自身的开发工作区

## 3. 内容对象契约

### 3.1 项目域对象（`03_Goal-Projects/`）

- `Target`：目标是什么、为什么值得做
- `Plan`：如何推进、阶段与里程碑
- `Cognition`：认知演进、复盘、洞见
- `Decision`：关键取舍与理由

### 3.2 证据对象（跨目录）

优先作为“可引用证据”的文档类型：

1. 认知 / 复盘 / 决策文档
2. 时间线与互动记录
3. README / index 导航文档

### 3.3 示例对象

公开模板允许保留示例对象，但需满足：

- 不泄露可反向识别的真实隐私
- 明确标识为 example / template / sanitized
- 不把私有真源仓的具体人、具体关系、具体敏感约束直接复制进来

## 4. 引用与回链规则

1. 逢引用必内链：引用仓库内内容时附相对路径
2. 先检索后判断：未检索前不要断言“没有”
3. 概念触发补检索：出现关键概念时补查复盘 / 决策 / 原则
4. 新增路径型引用后要做可达性校验

执行入口模板见：[`../AGENTS.md.template`](../AGENTS.md.template)

## 5. 与可选 indexing 的接口（候选）

本协议可给未来自研或外部 indexing 工具提供三类输入：

- 目录语义
- 对象类型边界
- 证据优先级

历史草案：

- [`indexing-contract.v1.md`](indexing-contract.v1.md)

## 6. 版本治理

1. 目录语义有变化时更新本文件
2. 若存在依赖本协议的候选工具，再同步更新其草案或脚本
3. 如变更较大，在 `../../develop/plans/` 留计划或说明

## 7. 真源与总结关系

1. 各内容域的局部协议文档是唯一真源
2. `00_Protocol` 负责跨域总结与消费接口，不复制真源正文
3. 总结层必须通过相对路径引用真源文档，确保可下钻与可校验
