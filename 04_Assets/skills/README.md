# skills

这里放你的技能画像与能力证据。

它解决的不是“你会什么”的口头描述，而是“哪些能力已经稳定到可以当成长期资产来讲”。

## 建议拆分维度

- Hard skills：技术栈 / 领域能力 / 工具链
- Soft skills：协作方式、沟通、写作、组织能力
- Methods：可复用的方法论与工作流

## 最小建议

- `skills.md`
  - 技能总表
  - 每项能力尽量附 1 条证据链接、代表作品或实际使用场景

## 边界

- 尚未稳定、仍依赖外部机会触发的能力线索，更适合先留在 `05_Resources/`
- 已能稳定复用、可被作品或经历支撑的能力，再沉淀到这里
