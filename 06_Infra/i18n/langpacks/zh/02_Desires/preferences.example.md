# Preferences / 偏好（Example）

> 说明：脱敏示例，仅用于展示结构与写法；请用你自己的版本替换。

## Work Style / 工作方式

- Prefer:
  - 小步快跑：先做最小闭环 Demo，再逐步抽象协议/模板。
  - Refactor > Append：更愿意把碎片重构成对象，而不是不断追加流水账。
  - 写下“下一步”：每次互动/思考最好落到可执行动作或明确“暂无动作”。
- Avoid:
  - 长期只做 PPT/对齐会议而无真实产出。
  - 目标含糊但压力很大（只要求“快”，不定义“对”）。

## Communication / 沟通偏好

- Prefer:
  - 先对齐目标与约束：一句话目标 + 3 条边界 + 1 个验收标准。
  - 低频但高质量：减少“随时 ping”，增加“成块的更新”。
- Avoid:
  - 需要读心的沟通：只给情绪，不给信息与上下文。

## Environment / 环境偏好

- Prefer:
  - 能长期积累：有机会把工作产出沉淀成可公开的资产（论文/开源/写作）。
  - 尊重边界：对休息与专注时间有基本尊重。
- Avoid:
  - 长期不确定性 + 没有对冲策略（没有可迁移产出、没有退出路径）。

## Tooling / 工具偏好

- Prefer:
  - Git + Markdown + 全文检索（search-first）。
  - Code Agent：用于整理、重构、归档，而不是只写聊天摘要。
- Avoid:
  - 把数据锁在单一平台的私有格式里。
