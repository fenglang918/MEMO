# 学术研究路径 vs 直接工作（Plan）

> 说明：脱敏示例项目。公开模板版只保留决策结构、验证动作与常见风险，不保留真实个人约束。

## 总体策略

- 先把两条路径的**本质区别**讲清楚，再进入信息收集与现实检验。
- 用“现实世界证据”替代纯脑内推演：访谈、试做、小规模交付、成本与退出路径检查。
- 最终输出一条 ADR，而不是停在“我好像想明白了”的口头状态。

## 阶段与里程碑

### 阶段 1：澄清框架（本周）

- [ ] 完成路径差异总纲：`cognition/001-phd-vs-job-essence.md`
- [ ] 列出需要验证的关键假设
- [ ] 补齐当前不可妥协项与可妥协项

### 阶段 2：信息收集（接下来 2–4 周）

- [ ] 访谈：至少 3 位学术路径相关者 / 3 位工业路径相关者
- [ ] 收集真实样本：日常节奏、评价机制、压力来源、退出成本、机会结构
- [ ] 记录样本偏差，避免只听“幸存者叙事”

### 阶段 3：现实试做（接下来 2–6 周）

- [ ] 研究试做：选一个 2–4 周可收敛的小研究问题
- [ ] 工程试做：选一个 2–4 周可交付的小工程或产品问题
- [ ] 记录体验差异：动机、心流、压力、反馈质量、可持续性

### 阶段 4：做出选择（截止日期：____）

- [ ] 写 ADR：`decisions/001-choose-path.md`
- [ ] 把结论落到下一步动作：申请、求职、作品集、沟通、时间表

## 常见风险与应对

- 风险：被“身份想象”带跑，而不是看真实日常
  - 应对：强制加入现实样本与试做记录
- 风险：信息来源过于单一
  - 应对：覆盖不同阶段、不同结果、不同组织生态
- 风险：拖延到窗口关闭
  - 应对：写死截止日期和最小可执行版本
- 风险：把大决策做成单次情绪反应
  - 应对：至少保留一轮“反事实检查”和一轮“隔天重看”
