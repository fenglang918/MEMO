# Reading Materials as Resources

- Date: 2026-01-01
- Status: `#status/active`

- Keywords: 阅读材料；资源库；信息摄入；执行干扰
- Tags: `#area/research` `#area/time` `#scene/information-intake` `#trigger/context-switch` `#scope/personal` `#status/active`
- Trigger signals:
  - 读到一篇好文章就想立刻把它塞进当前项目
  - `01_Inbox/` 里积压了很多“应该看”的 PDF 和长文
  - 阅读任务正在干扰高压执行期的主线推进
- Decision rule:
  - **待阅读材料先作为资源进入 `05_Resources/`，不要直接塞进高压项目或被误写成目标。**
  - 只有当某个项目中的具体行动需要它时，再把它从资源库唤醒并转化为项目认知或资产。

## Context

- 阅读材料在你还没消化、也没有落地产出之前，本质上只是 potential，不是行动本身。
- 如果把新材料直接塞进当前项目，容易制造“我是不是偏题了”的额外压力。
- 如果只是堆在 `01_Inbox/`，又会带来“迟迟没有读完”的负罪感。

## Working Loop

1. 收容：把文章、PDF、资料包放到 `05_Resources/` 对应目录。
2. 蛰伏：只保留极简 README，说明材料是什么、以后可能在哪些项目里有用。
3. 唤醒：当具体项目需要时，再从资源库里抽出并深读。
4. 转化：把消化后的结论写进项目 `cognition/`，或在确认复用后抽象成 `04_Assets/`。

## Examples

- 正例：
  - 看到一批关于 personal CRM 或时间管理的文章，先放进 `../05_Resources/reading/`，等真正整理 network 系统或 operations 流程时再集中消化。
- 正例：
  - 下载一组研究报告，先写 3 行 README 说明“它们适合在做 proposal 或 related work 时调取”，而不是立即开启“全面总结”。
