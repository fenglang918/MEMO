# Module Instructions: network

## 目录定位

- 本目录用于维护“按人组织、为检索和行动服务”的人脉资源库。
- 这里放运行中的联系人卡片、检索说明、可选 CLI 和架构说明。
- 不在这里放完整 CRM 平台设计、项目级协作 ADR，或与 network 无关的泛化方法论。

## 读写顺序

- 先看 `index.md`：获取目录结构、标签体系、检索方法与维护节奏。
- 新增联系人时，先从 `people/_template.md` 建卡，再补最小字段。
- 需要理解设计动机时，再看 `requirements-architecture.md`。
- 需要 Agent workflow 时，再看 `../../.agents/skills/me-network-crm/SKILL.md`。

## 新增与修改约束

- 联系人卡片默认一人一文件，优先放在 `people/` 下。
- 每张卡片至少写清：`Handle/ID`、`关键词`、`Tags`、`状态`、`最后联系`、`下次跟进`。
- tags 先复用 `index.md` 里的建议前缀；确需新增时，先更新 `index.md` 再落到卡片。
- 联系方式只保留“定位信息”，不要记录可直接滥用的明文敏感信息。
- 公开模板里的示例卡片必须保持脱敏、不可反向识别。

## 与其他内容域的关系

- `network` 回答的是“这个人是谁、我为什么会再找 TA、TA 能提供什么资源”。
- 如果某个人只在某个项目语境下重要，应优先在项目里记录当前上下文，再按需回链到 network 主卡。
- 如果某个输入仍然是原始截图、聊天或资料包，先放 `01_Inbox/`，再决定是否入人脉库。
