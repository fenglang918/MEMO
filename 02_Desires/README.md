# Desires

这里放“我想要什么 / 我不想要什么 / 我在乎什么”的上游驱动层（公开模板版）。

建议把它视为 `03_Goal-Projects/` 的来源，而不是把“目标”直接写成项目。

它回答的不是“我现在在做什么”，而是：

- 我长期真正想要什么
- 我不想要什么
- 我在重要决策里默认看重什么
- 我想成为怎样的人

推荐最小集合（可按需扩展）：

- `values.md`：价值观与拒绝原则（非目标/红线）
- `preferences.md`：偏好与反偏好（工作/生活/协作方式）
- `desires.md`：愿望池（按主题分组 + 优先级 + 最近复查）
- `RoleModels/`（可选进阶层）：偶像 / 标杆 / 参照系

本模板同时提供 `*.example.md` 作为脱敏示例（只用于演示写法与结构）。

## 和其他层的关系

- `02_Desires/` 提供上游驱动力
- `07_Principles/` 提供跨项目裁决规则
- `03_Goal-Projects/` 负责把驱动力对象化成项目与计划

一句话：

`Desires` 讲“想要什么”，`Principles` 讲“怎么裁决”，`Goal-Projects` 讲“具体做什么”。
