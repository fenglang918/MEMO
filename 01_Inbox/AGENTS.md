# Module Instructions: 01_Inbox

## 目录定位

- 本目录用于承接尚未归位的输入。
- 这里可以放原始文件、半成品材料、讨论缓冲与待抽取任务的输入。
- 它是模板里的缓冲层，不是长期稳定真源。

## 读写顺序

- 先看 `README.md`：理解导流方向与典型 workflow。
- 如果要保留原始证据，再看 `raw/README.md`。
- 如果已经处理过但仍是中间态，再看 `processed/README.md`。
- 需要导流归档时，看 `../.agents/skills/inbox-folder-archiver/SKILL.md`。

## 新增与修改约束

- `01_Inbox/` 里的内容应尽量保持“短暂停留”，不要长期堆积成稳定真源。
- 如果输入已经足够清楚，应尽快导流到 `03_Goal-Projects/`、`04_Assets/`、`05_Resources/` 或 `08_Operations/`。
- AI-native workflow 下，不强制所有输入先落到 `raw/`；只有需要证据与回溯时才保留原始材料。
- 公开模板中的示例和占位内容应保持通用，不放真实私有输入。
